# Решение проблем

## Ошибка «Failed to fetch»

Обычно возникает при запросах к Appwrite (авторизация, список событий и т.д.), когда браузер или сервер не может выполнить запрос к API.

### Возможные причины и что проверить

1. **Неверные или пустые переменные окружения**
   - В `.env.local` должны быть заданы: `NEXT_PUBLIC_APPWRITE_ENDPOINT`, `NEXT_PUBLIC_APPWRITE_PROJECT_ID`, `NEXT_PUBLIC_APPWRITE_DATABASE_ID`, `NEXT_PUBLIC_APPWRITE_STORAGE_ID`, а также ID коллекций (или дефолты после `db:setup`).
   - Не должно быть плейсхолдеров вида `your_project_id_here`.
   - После изменения `.env.local` перезапустите `npm run dev`.

2. **Сеть и доступность Appwrite**
   - Проверьте в браузере: открывается ли `https://cloud.appwrite.io/v1/health/version` (или ваш endpoint + `/health/version`).
   - Убедитесь, что фаервол/прокси не блокируют запросы к Appwrite.

3. **CORS**
   - Для облачного Appwrite CORS обычно уже настроен. Если используете свой хост — добавьте домен приложения в настройки проекта в Appwrite Console.

4. **Поведение в приложении**
   - В коде добавлена обработка сетевых ошибок: при «Failed to fetch» сервисы возвращают безопасные значения (пустой список событий, `null` для пользователя и т.д.), чтобы интерфейс не падал. В консоли браузера при этом может появляться предупреждение о недоступности сети/Appwrite.

## Пользователь не может войти

- Проверьте, что в Appwrite включён провайдер Auth (например, Email/Password).
- Для ролей USER и ORGANIZER в коллекции `users` должно быть `isActive: true`. Иначе после входа пользователь получит сообщение об ожидании активации и будет разлогинен. Админы создаются с `isActive: true`.

## Коллекции не создаются (db:setup)

- Убедитесь, что в `.env.local` задан `APPWRITE_API_KEY` с правами на создание коллекций и индексов в выбранной базе.
- Проверьте `NEXT_PUBLIC_APPWRITE_DATABASE_ID` — база должна существовать в проекте.
- Запустите `npm run db:test` и посмотрите текст ошибки.

## Пустой каталог событий

- Выполните `npm run db:setup`, чтобы коллекция `events` и индексы существовали.
- В списке отображаются только события со статусом `PUBLISHED`. Черновики видны только организатору и админу в соответствующих разделах.
- При «Failed to fetch» приложение намеренно возвращает пустой список; исправьте доступ к Appwrite (см. выше).

## Ошибки линтера или TypeScript

- Выполните `npm run lint`.
- Проверьте версии Node (рекомендуется 20+) и установку зависимостей: `npm ci` или заново `npm install`.

## Полезные команды

```bash
npm run db:test      # Проверка подключения к Appwrite
npm run db:setup     # Создание/обновление коллекций
npm run lint        # Проверка кода
```
