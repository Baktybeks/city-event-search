# Скрипты и база данных

## NPM-скрипты

| Команда | Описание |
|---------|----------|
| `npm run dev` | Запуск в режиме разработки (Next.js dev server) |
| `npm run build` | Сборка для production |
| `npm run start` | Запуск production-сборки |
| `npm run lint` | Запуск линтера |
| `npm run db:test` | Проверка подключения к Appwrite |
| `npm run db:setup` | Создание коллекций и индексов (без удаления существующих) |
| `npm run db:reset` | Удаление коллекций (**данные теряются**) |
| `npm run db:reset-setup` | Сброс коллекций и повторное создание (reset + setup) |
| `npm run db:deleteAttribute` | Удаление атрибута (использование: см. скрипт) |

Скрипты БД читают переменные из `.env.local` (через `dotenv`). Нужны `NEXT_PUBLIC_APPWRITE_*` и `APPWRITE_API_KEY`.

## setupCollections.js

Файл: `scripts/setupCollections.js`.

### Режимы работы

Запуск с аргументом:

```bash
node scripts/setupCollections.js <command>
```

| command | Действие |
|---------|----------|
| `setup` | Создать недостающие коллекции, атрибуты и индексы. Существующие данные не удаляются. |
| `reset` | Удалить все коллекции, описанные в конфиге. |
| `reset-setup` | Выполнить `reset`, затем `setup`. |
| `deleteAttribute` | Удалить указанный атрибут из коллекции (параметры задаются в коде/аргументах скрипта). |

Без аргумента скрипт может выводить справку или выполнять `setup` — см. код в начале файла.

### Что создаётся при setup

1. **Коллекции**: users, events, registrations, favorites, views.
2. **Атрибуты**: по схемам в `COLLECTION_SCHEMAS` (типы, обязательность, размеры, enum-значения).
3. **Индексы**: по `COLLECTION_INDEXES` (в т.ч. уникальные пары для registrations и favorites).
4. **Права доступа**: при необходимости скрипт настраивает permissions для чтения/записи (см. скрипт).

Идентификаторы коллекций берутся из переменных окружения (или дефолты: `users`, `events`, …). Имена коллекций в Appwrite могут совпадать с этими ID или быть заданы при создании.

### Проверка подключения

```bash
npm run db:test
```

Скрипт `scripts/testConnection.js` подключается к Appwrite с теми же переменными из `.env.local` и проверяет доступ. При успехе в консоли будет соответствующее сообщение; при ошибке — текст ошибки (сеть, неверный ключ, неверный project/database ID и т.д.).

## Рекомендации

- Перед первым запуском приложения выполните `npm run db:test`, затем `npm run db:setup`.
- Не используйте `db:reset` и `db:reset-setup` на production без необходимости — все данные в этих коллекциях будут удалены.
- После смены проекта Appwrite или базы обновите `.env.local` и при необходимости заново выполните `db:setup` (или `db:reset-setup` для чистой базы).
